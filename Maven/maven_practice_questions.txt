- Превратить обычный проект в мавен проект без пересоздания проекта (Add framework support idea function)
	ПКМ по проекту -> Add Framework Support... -> Maven

- Добавить новый мавен модуль внутрь существующего проекта с именем темы "Maven"(Каждая новая тема будет отдельным модулем)
	ПКМ по проекту -> New -> Module... -> Maven -> (выбрать archetype) -> Next -> 'написать имя модуля' -> Finish

- Посмотреть как изменился Родительский и Дочерний модуль
	В родительском pom файле добавилось:
		<packaging>pom</packaging>
		и
		 <modules>
		        <module>untitled</module>
		 </modules>

	в созданом модуле
	GAV параметры родительского модуля обёрнуты в тег <parent> и объявлен свой тег <artifactId> - артифакт модуля

- Внутри мавен проекта посмотреть как расположились папки, почему так?
	1) при создании maven проекта и(или) модуля, автоматически создается стандартная maven-структура каталогов, в которых будет хранится сорс код, ресурсы, тесты
	2) IDE помечает проект и модуль Maven
	3) в модулях создается файл : <название модуля>.iml (пока не понял для чего он, и это файл принадлежит maven или idea)
